# Bug Report Samples

A collection of professional, well-structured defect reports demonstrating clear bug documentation — reproduction steps, severity, priority, and expected vs. actual behavior.

All reports are based on **fictional applications** with original content. No proprietary or company data is included.

---

## Why This Matters

A bug report is only useful if a developer can reproduce and fix the issue without asking questions. These samples show how I document defects: clearly, completely, and consistently.

## Anatomy of a Good Bug Report

Every report in this repository includes:

- **Unique ID & title** — short and descriptive
- **Environment** — where it was found
- **Severity & priority** — impact vs. urgency
- **Preconditions** — required setup
- **Steps to reproduce** — numbered and repeatable
- **Expected vs. actual result** — the core of the defect
- **Evidence** — where a screenshot/log would be attached
